Complete Flow Example

User calls:

POST /api/v1/auth/change-password

with valid JWT.

Flask executes:

1. before_request()
      ↓
logs:
POST /api/v1/auth/change-password

2. jwt_required()
      ↓
token verified

3. change_password()
      ↓
password updated

4. logging.info(
       "User admin changed password"
   )
      ↓
writes business log

5. response returned

Final app.log:

2026-06-17 11:00:00 - INFO - POST /api/v1/auth/change-password
2026-06-17 11:00:00 - INFO - User admin changed password

So:

@app.before_request = automatic Flask hook, not a route.
No permission is needed to create it.
It runs before every request.
logging.info(...) simply writes messages to the log file.
JWT verification still happens separately through @jwt_required().
